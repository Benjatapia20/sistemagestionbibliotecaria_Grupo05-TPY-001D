Sistema Bibliotecario Resiliente

Plataforma de gestión bibliotecaria diseñada para seguir funcionando sin conexión a internet, evitando atrasos, multas injustas y dependencia de servicios externos.

🏛️ Local-First 🔄 Offline-First ☁️ Sincronización Online 💳 Pagos Integrados 🤖 Chatbot IA

🎯 El Problema

Contexto real que motivó el desarrollo de este sistema

⚠ Situación detectada

En una biblioteca real, su sistema de gestión dependía completamente de internet. Cuando el servicio de internet fallaba —algo frecuente en ciertas zonas— el sistema completo dejaba de funcionar. Esto provocaba:

  • Devoluciones no registradas → multas por atraso injustas
  • Préstamos bloqueados → usuarios sin acceso a libros
  • Pérdida de datos → transacciones no guardadas
  • Dependencia total de un solo punto de fallo

✅ La Solución

Un sistema local-first que opera con base de datos PostgreSQL en Docker. Funciona completamente offline dentro de la red local de la biblioteca. Cuando hay internet, se sincroniza automáticamente con Supabase para permitir consultas online (catálogo, disponibilidad, multas). La experiencia del usuario es idéntica en ambos modos.

🗄️ Base de Datos

Schema api — 22 tablas con triggers, funciones y constraints

📚 Tablas principales

  • autores — nombre, nacionalidad, biografía
  • libros — título, sinopsis, ISBN, carátula
  • ejemplares — código LIB-X-XXX, estado, condición
  • editoriales, géneros, etiquetas
  • libros_generos, libros_etiquetas (N:M)

👤 Usuarios y préstamos

  • usuario — UUID, nombre_usuario, rol, RUT, email
  • prestamos — 9 estados: pendiente → en_revision → por_entregar → activo → devuelto
  • prestamos_grupo — pedidos con código PED-YYYY-XXXX
  • multas — $500/día, máximo $10.000, pagos Flow

📋 Reseñas y listas

  • resenas — puntuación 1-5, comentario, fotos
  • listas — Favoritos, Por leer, Leídos (default)
  • listas_libros — relación N:M
  • Trigger: solo reseñar libros devueltos

⚡ Triggers (11)

  • updated_at — timestamp automático en todas las tablas
  • codigo_ejemplar — genera LIB-X-XXX
  • validar_ejemplar_prestamo — gestiona estados (disponible/reservado/prestado)
  • actualizar_disponibilidad_libro — recalcula stock
  • marcar_atrasado_y_multar — multas automáticas
  • validar_resena — solo si hay préstamo devuelto
  • generar_codigo_pedido — PED-YYYY-XXXX

🔄 Flujos Principales

Los procesos más importantes del sistema

📖 Préstamo de libros

1

Catálogo

Usuario explora libros con búsqueda, filtros y portadas

2

Carrito

Agrega hasta 5 ejemplares, elige fecha de devolución

3

Pedido

Se crea grupo con estado pendiente + código PED

4

Revisión

Admin asigna ejemplares, marca no disponibles si aplica

5

Entrega

Préstamo activo, ejemplar → prestado, QR generado

6

Devolución

Escaneo QR, ejemplar liberado, opción de reseña

💸 Multas y Pagos

1

Vencimiento

Trigger marca activo → atrasado si fecha < hoy

2

Multa

Se crea registro con $500/día (máx $10.000)

3

Notificación

Badge rojo en header: "Pagar $X.XXX"

4

Pago

Flow.cl (Webpay) o Simular pago, PDF comprobante

5

Desbloqueo

Multas pagadas → usuario puede pedir libros de nuevo

☁️ Sincronización

1

Local

Todo funciona con PostgreSQL Docker

2

Sync Push

UPSERT incremental: solo cambios desde última sync

3

Supabase

Réplica online con las mismas tablas

4

Netlify

Frontend online consulta Supabase (catálogo, perfil)

5

Webhook

Netlify Functions firma pagos Flow online

✨ Características

Funcionalidades implementadas

🔐 Autenticación

Login, registro, recuperación de contraseña. 3 roles: admin, bibliotecario, usuario. Persistencia en localStorage.

📚 Catálogo

57 libros con 37 autores. Búsqueda por título, filtro por género/autor/etiqueta. Portadas de OpenLibrary + Google Books. Stock en tiempo real.

🛒 Carrito + Pedidos

Máximo 5 ejemplares. Date picker para devolución. Carrito en PostgreSQL. Pedidos con código PED-YYYY-XXXX.

👤 Perfiles públicos

URL /perfil/{username}. Avatar, bio, listas públicas, reseñas, libros leídos. Tabs con variante line. Edición inline.

⭐ Reseñas

Puntuación 1-5 estrellas, comentarios, fotos con lightbox. Solo usuarios que devolvieron el libro. Editar/eliminar.

📋 Listas

Favoritos, Por leer, Leídos. Marcar como leído quita de Por leer. Contador en perfil. Portada, visibilidad pública/privada.

🏛️ Admin

CRUD de libros, autores, géneros, editoriales, etiquetas, usuarios. Toggle disponibilidad. QR scanner. Dashboard con estadísticas.

🔍 Operaciones

Revisión de pedidos con asignación de ejemplares. Toggle modo online/offline. Búsqueda de usuarios activos. Notificaciones push.

💳 Multas y Flow

Multas por atraso ($500/día, máx $10.000). Badge rojo en header. Pago con Flow.cl (Webpay sandbox). PDF de comprobante. Bloqueo de préstamos.

🤖 Chatbot IA

BiblioBot con Qwen 2.5 14B vía Ollama. Function calling: buscar libros, autores, géneros, mis préstamos. Agregar al carrito por voz/texto.

🗺️ Mapa

Mapa interactivo de la biblioteca con 23 secciones: Atención, Lectura, Libros, WiFi, Baños. Imagen personalizable.

📱 QR + PDF

Códigos QR en ejemplares (LIB-X-XXX). PDF de comprobante de pedido con QR. Escáner QR para asignar ejemplares.

🔄 Máquina de Estados

Transiciones de préstamos y ejemplares

📖 Préstamo

pendiente → en_revision → por_entregar → activo → devuelto
                 ↘ no_disponible → solicita_aprobacion
                 ↘ rechazado
activo → atrasado (automático si fecha < hoy)

📦 Ejemplar

Estado (disponibilidad): disponible → reservado → prestado → disponible
Condición (física): bueno, regular, dañado, perdido

Transiciones automáticas vía triggers. Stock = COUNT(estado='disponible')
